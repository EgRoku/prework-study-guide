HTML

  • The head element contains informationabout the webpage
  • The body element represents the visible content shown to the user
  • Sections help organize related topics under its own category
  • Li makes bullet points to organize info
  • Headers should go in ascending order to make assistance devices make sense
  • Ul creates the unordered list of items while Ol makes an ordered list
  • Semantic HTML elements indicate the role of their content, as well as give structure to the content of the webpage

CSS

  • Inline CSS is used when we want to style an element directly in the HTML file
  • An internal style sheet contains CSS rules for the head section of the HTML file
  • External CSS is the most common way to style a webpage because it allows us to keep all CSS rules in a seperate file
  • Seperation of concerns is a term used to refer to seperating code into different files based on use
  • a selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A margin property indicates how much space we want on the outside of an element; margin property of 0 indicates NO extra space
  • The padding property adds space around the content inside the element; not to be confused with margin
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained. DRY can greatly reduce the overall size and complexity of a codebase.
  • Link element creates an extension to an external file
  • The rel attribute specifies the relationship between the current doc and the linked doc
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

GIT

  • git status: checks what branch we are currently in
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A commits all files in the working branch
  • -m flag associates a message with the code
  • git checkout 'branch name' to navigate to said branch
  • git pull origin to pull in the latest changes
  • git push 'branch name' to push to repository

JAVASCRIPT

  • Javascript is what makes a static HTML/CSS webpage interactive
  • Variables are containers that store values
  • A semicolon at the end of a kine indicates where a statement ends. It is only needed when you need to seperate statements on a single line
  • Comments are snippets of text that can be added alongside code. Use /* Everything between is a comment. */ "//this is also a comment"
  • An operator is a mathematical symbol that produces a result based on two values aka variables
  • Conditionals are code structures used to test wether an expression returns true or not
  • Functions are a way of packing functions you wish to reuse
  • Events are code structures that listen for activity in the browser and run code as a response
  • An Array is a single variable that is used to hold a group of data
  • A function is different from a for loop because a function does not auto execute when the file is ran
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
